Repaso¶

  • Los mtodos de aprendizaje no supervisado apuntan realizar un anlisis exploratorio de los datos, para poder descubrir aspectos de inters sobre los mismos dejando de lado su posible asociacin con una respuesta $y$.
  • El conjunto de datos que se analiza es la matriz $X$, con el fin de extraer modelos o conocimiento sobre los datos de la misma. Debido a que no incluimos un conjunto $y$ de salidas para aprender iterativamente, para esta tarea se utilizan tcnicas distintas.
  • No obstante, para ciertos problemas es posible combinar el conocimiento extrado a partir de este tipo de aprendizaje para mejorar los modelos de aprendizaje supervisado vistos anteriormente.
  • Algunos ejemplos de tarea de aprendizaje no supervisado:
    • Dada una coleccin de imgenes, agrupar aquellas similares.
    • Dadas varias fuentes de sonido en una misma pista de audio, como por ejemplo una persona hablando por telfono y un parlante emitiendo msica, obtener la separacin de las distintas fuentes.
    • Entender cmo se correlacionan los features de un determinado conjunto de datos.
  • En la presente clase vamos a ver la implementacin de dos tcnicas muy utilizadas: clstering y PCA.

Análisis de componentes principales (PCA)¶

  • En la prctica, los features $X$ de un dataset suelen tener un grado de relacin entre s (ej. pensar en features edad y altura para una muestra de datos de nios y adolescentes).
  • Como muchas veces, para estos datasets con datos correlacionados, es baja la cantidad de features que definen al dataset, debera ser posible encontrar la forma de explicar la mayor parte posible del dataset con la menor cantidad de features posible.
  • El anlisis de componentes principales (PCA) realiza esto al calcular features transformados que surgen como combinacin lineal de los dems, permitiendo as visualizar esta correlacin y reducir la dimensin del dataset y por ende el efecto de la Curse of Dimensionality.
  • En otras palabras, PCA apunta a reducir la dimensionalidad de un dataset al considerar qu features explican mejor la varianza de los datos.
  • Para ello se crea una proyeccin linear del dataset. Los datos correlacionados se obtienen mediante un nmero menor de variables representativas no correlacionadas (ortogonales) que expliquen de forma conjunta la varianza del set original.

Repaso rápido:¶

Repaso rpido de lgebra

Objetivo: encontrar una matriz de vectores de pesos/coeficientes $W(p,p)$ (cuyos vectores columna se conocen como componentes principales) que transforme a los datos $X(n,p)$ (normalizados con media 0 en cada columna) tal que la nueva transformacin maximice la varianza de los datos, mediante

$T = XW$

Cada fila de $T$ representa el mapeo de la correspondiente fila de $X$ para el primer, segundo y ltimo componente principal considerado (hasta un mximo de $min(n,p)$), mientras que cada columna representa los valores de cada observacin para el primer componente principal.

En $W$, cada columna/componente tiene los pesos del j-simo componente principal (con un mximo de $min(n,p)$).

autoval, autovec = np.linalg.eig(cov_X)  # descomponemos

Si utilizamos PCA para un conjunto $X_{train}$ y hacemos un modelo para aproximar el mismo, qu hacemos si tenemos nuevos datos de test $X_{test}$?

  • Para los nuevos datos tenemos primero que normalizarlos con media 0, restndole a cada feature la media del feature para el conjunto de entrenamiento (puede tambin normalizarse en cuanto a la desviacin estndar, z-score).
  • Tras eso, deben ser transformados con respecto a los componentes principales del conjunto de entrenamiento con pca.transform($X{test}$)_. A partir de ah ya pueden ser utilizados con el modelo empleado para el conjunto $X$.

Clustering¶

  • Clustering se refiere a aquellos mtodos que separan un conjunto de datos $X$ en subgrupos (clsters), de tal forma que los datos pertenecientes a cada grupo tengan entre s la mayor relacin posible.
  • Dado que no consideramos las etiquetas $y$, estos grupos se infieren exclusivamente a partir de los features de nuestra matriz de datos $X$.
  • Para entender clustering, es posible interpretarlo como un mtodo que le asigna un vector similar a un label a cada observacin de $X$, donde cada una de las ocurrencias de dicho vector representa el clster de la correspondiente observacin.

K-Means Clustering¶

K-Means Clustering es un mtodo iterativo y de propsito general muy simple que particiona las observaciones en un nmero predefinido $K$ de clsters, donde cada dato se asigna a un clster. El algoritmo consiste en;

  1. Cada dato se asigna al clster ms cercano.
  2. Se modifica el centroide de los clsters respecto a los datos asignados.
  3. Se repiten 1. y 2. hasta que no haya cambios.
  • Veamos cmo funciona el algoritmo K-Means en el iris dataset.
  • Este es un ejemplo en el cul no tiene mucho sentido hacer clstering debido a que tenemos los labels $y$, pero dado que es un conjunto conocido vamos a ver cmo convergen los distintos clsters.
In [23]:
from sklearn.cluster import KMeans

def graficar_kmeans(number_of_cluster):
    
    kmeans = KMeans(n_clusters=number_of_cluster, random_state=80)
    kmeans.fit(X)

    print("Nmero de clster asignado a cada observacin ", kmeans.labels_)
    print("Centroides de cada clster ", kmeans.cluster_centers_)

    for i in range(np.max(kmeans.labels_)+1):
        plt.plot(X[kmeans.labels_==i,0], X[kmeans.labels_==i,1], 's', label="Cluster " + str(i+1))
        plt.plot(kmeans.cluster_centers_[i,0], kmeans.cluster_centers_[i,1], 'o', color='orange')

    plt.xlabel(iris.feature_names[0])
    plt.ylabel(iris.feature_names[1])
    plt.legend(loc='best', numpoints=1)
    plt.show()

Mixture Models¶

En los Mixture Models, se asume que los datos generados vienen de una mezcla (mixture) de distribuciones probabilsticas de la misma familia (ej: Gaussianas) pero con distintos parmetros. En nuestro ejemplo nos vamos a centrar en Mixture of Gaussians.

  • Mixture of Gaussians puede ser pensado simplificadamente como una "generalizacin del KMeans" en donde se asume que los centroides son formados a partir de distribuciones normales, adems de incorporar informacin sobre la varianza de los datos.
  • Veamos cmo los mismos logran separar mucho mejor los datos de nuestro ltimo ejemplo.

Conclusiones de la introducción al aprendizaje no supervisado¶

  • El aprendizaje no supervisado es una forma de aprendizaje donde el foco est centrado en extraer conocimiento sobre los datos con los que contamos, sin considerar la salida de los mismos.
  • Hemos visto los dos enfoques principales: PCA y clustering.
  • Con PCA, tomamos un dataset y, a partir de una transformacin lineal, vemos cun correlacionados estn sus datos al contar con vectores ortogonales que los transforman.
  • Con clustering, lo que hacemos es ver cmo los datos pueden dividirse en subgrupos, de modo tal que podamos entender mejor las propiedades subyacentes de los features.

Ambos enfoques nos sirven para obtener conocimiento desde una perspectiva distinta de la propuesta por las tcnicas de aprendizaje supervisado.
